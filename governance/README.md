# IT Governance

This directory contains policies, standards, procedures, and audit documentation.

## Structure

### `policies/`
IT Policies and Guidelines:
- `security_policy.md` - Information Security Policy
- `data_retention_policy.md` - Data Retention Policy
- `acceptable_use_policy.md` - Acceptable Use Policy
- `incident_response_policy.md` - Incident Response Policy
- `backup_policy.md` - Backup and Recovery Policy

### `standards/`
Technical Standards:
- `coding_standards.md` - Code Quality Standards
- `api_standards.md` - API Design Standards
- `security_standards.md` - Security Standards
- `documentation_standards.md` - Documentation Standards
- `testing_standards.md` - Testing Standards

### `procedures/`
Standard Operating Procedures (SOPs):
- `deployment_procedure.md` - Deployment SOP
- `backup_procedure.md` - Backup & Restore SOP
- `incident_handling.md` - Incident Response SOP
- `change_management.md` - Change Management SOP
- `security_incident.md` - Security Incident Handling

### `audits/`
Audit Logs and Reports:
- `internal_audits/` - Internal audit reports
- `external_audits/` - External audit reports
- `compliance_audits/` - Compliance audit results
- `security_audits/` - Security audit logs

## Governance Framework

### Policy Review Cycle
- **Quarterly**: Review and update as needed
- **Annually**: Comprehensive policy review
- **Ad-hoc**: Emergency policy updates

### Compliance Requirements
- ✅ Data Protection Act (Cambodia)
- ✅ Government IT Security Standards
- ✅ National Digital Infrastructure Policy
- ✅ ISO 27001 Alignment
- ✅ ITIL Best Practices

### Audit Schedule
- **Internal Audits**: Quarterly
- **External Audits**: Annually
- **Security Audits**: Bi-annually
- **Compliance Audits**: Annually

## Access Control

- **Policies**: Read access for all staff, Write access for governance team
- **Standards**: Read access for technical team
- **Procedures**: Read access for operations team
- **Audits**: Restricted access (Management, Auditors, Compliance Officers)

## Classification

- **Level**: Confidential
- **Access**: Management, Compliance Officers, Auditors
- **Backup**: Monthly
- **Retention**: Permanent (policies), 7 years (audits)

## Document Control

All governance documents must include:
- Version number
- Last updated date
- Approval authority
- Next review date
- Change history
